# Porting Chrome Authenticator → Capacitor Mobile App

This plan ports the existing Chrome Extension (Vue 3 + Webpack) into a fully native iOS & Android app using **Capacitor.js + Vite**. The core TOTP logic, all Vue components, and SCSS styles are reused with minimal changes. Only the Chrome-specific API layer is replaced.

## User Review Required

> [!IMPORTANT]
> **New project location**: The Capacitor app will be created as a **new sibling directory** at:
> `/Users/sociolla/Documents/ai-helper/mobile-authenticator-app/`
>
> The original Chrome extension at `mobile-authenticator/` will remain **untouched**.

> [!WARNING]
> **iOS build requires a Mac with Xcode installed.** Android requires Android Studio.
> Running `npx cap open ios` / `npx cap open android` will be left for you to do after setup, since it requires Xcode/Android Studio.

> [!IMPORTANT]
> **QR Scanner behavior change**: On mobile, the "Scan from current page" (Chrome tab feature) is replaced with **native camera QR scanning** via `@capacitor-mlkit/barcode-scanning`. This is actually a *better* UX — point camera at a QR code and it reads instantly.

### Phase 2 — Adapt Storage Layer

#### [MODIFY] `src/lib/storage.js`
**The single most critical change.** Replace all `chrome.storage.local.*` calls with `@capacitor/preferences`.

```diff
-import chrome API
+import { Preferences } from '@capacitor/preferences';

 export async function getAccounts() {
-  const data = await chrome.storage.local.get('accounts');
-  return data['accounts'] || [];
+  const { value } = await Preferences.get({ key: 'accounts' });
+  return value ? JSON.parse(value) : [];
 }

 export async function saveAccount(account) {
   const list = await getAccounts();
   // ... same logic ...
-  await chrome.storage.local.set({ accounts: list });
+  await Preferences.set({ key: 'accounts', value: JSON.stringify(list) });
 }
```
Same pattern for `updateAccount`, `deleteAccount`, `clearAllData`.

#### [MODIFY] `src/components/QrScanner.vue`
This is the most significant component change. Two modes:

1. **Image upload** (keep as-is — works on mobile via `<input type="file">`)
2. **"Scan from page"** → replaced with **"Use Camera"** button using `@capacitor-mlkit/barcode-scanning`

```diff
-async function scanFromPage() {
-  // chrome.tabs.query / chrome.storage.session ...
-}
+async function scanFromCamera() {
+  const { BarcodeScanner } = await import('@capacitor-mlkit/barcode-scanning');
+  const { barcodes } = await BarcodeScanner.scan({
+    formats: [BarcodeFormat.QrCode],
+  });
+  if (barcodes.length > 0) {
+    await processQrData(barcodes[0].rawValue);
+  }
+}
```

UI label changes: "Scan QR Code on Current Page" → **"Scan with Camera"**
Tip text: Remove Chrome context-menu tip.

- `max-height: 600px` → `100dvh` (dynamic viewport height)
